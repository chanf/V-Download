# 视频下载方法说明（含 B 站“只下载一半”专项）

## 1. 目标

本文档用于说明一个稳定的视频下载实现方法，并重点解决 B 站常见问题：

- 文件下载完成但时长只有一半；
- 只能播放前半段；
- 看起来“成功”但实际上内容不完整。

---

## 2. 通用下载流程（建议标准实现）

### 2.1 链接归一化

1. 从用户输入或分享文本中提取第一个有效 `http/https` 链接。
2. 去除结尾噪声字符（中文句号、逗号、右括号等）。
3. 对短链做重定向展开，拿到最终目标链接。

### 2.2 页面解析与 API 回退

1. 先抓页面 HTML（带正确 `User-Agent`、`Referer`、Cookies）。
2. 先从结构化字段提取直链（JSON 中的 `url/playAddr/downloadAddr/...`）。
3. 页面未命中时，回退到平台公开接口或兼容接口。
4. 若检测到明确受限状态（鉴权、审核、区域限制），返回“可读错误”，不要误报为“解析失败”。

### 2.3 下载执行

1. 先发起探测请求，拿到：
   - 最终 URL；
   - `Content-Type`；
   - `Content-Length`；
   - 是否支持 `Range`。
2. 大文件走分段下载，小文件走单线程下载。
3. 统一重试策略：
   - 网络错误/5xx/超时可重试；
   - 4xx 只对可恢复场景重试（例如签名过期后重新解析）。

### 2.4 完整性校验（必须）

下载结束后必须做校验，至少包含：

1. 若有 `Content-Length`，校验 `bytesWritten == Content-Length`。
2. MIME 与扩展名匹配（`video/*`）。
3. 可选增强：用 `ffprobe` 校验容器可读、时长大于最小阈值。

未通过校验必须判定失败，不可当成功。

## 3. B 站“只下载一半”问题：根因与修复

## 3.1 常见根因

1. **只下载了 `durl` 的首段**
   - 部分 B 站资源是多段 `durl`，只拿第 1 段会导致半截/不完整。

2. **拿到 DASH 的视频流，但没合并音频**
   - DASH 常见为视频流和音频流分离，只保存视频流会出现“看起来不完整”或体验异常。

3. **分段下载合并错误**
   - `Range` 边界计算错误、写入偏移错位、临时分片丢失，都会导致最终文件缺块。

4. **下载中途链接过期/风控中断**
   - B 站签名 URL 失效、Referer/Cookie 不完整，可能下载到一半返回 403/HTML。

5. **仅以“进度到 100%”判成功**
   - 没有做字节完整性校验，会把半截文件当成功。

## 3.2 必做修复策略

1. **`durl` 模式：全段下载并顺序拼接**
   - 按 `order` 下载每一段，全部成功后再串联输出。

2. **DASH 模式：同时下载音频与视频并合并**
   - 通过 `ffmpeg -c copy` 无损封装合并。

3. **每段独立重试 + 断点续传**
   - 失败段只重下失败片段，避免整文件重来。

4. **强制完整性校验**
   - 若有期望字节数必须严格对齐；
   - 合并后再验文件可读性与时长。

5. **请求头与 Cookie 对齐**
   - 固定 `Referer: https://www.bilibili.com/`；
   - 必要时带有效 Cookies（含 `SESSDATA` 场景）。

6. **检测到 HTML 响应立即失败并重解析**
   - 避免把鉴权页写进视频文件。

## 4. 推荐日志字段（便于快速定位“半截”问题）

每次下载建议记录：

1. 原始链接、解析后直链、最终请求 URL。
2. 平台类型、解析分支（页面/接口/durl/DASH）。
3. `Content-Type`、`Content-Length`、是否 `Range`。
4. 分段信息（段数、每段起止、每段重试次数）。
5. 实际写入字节、期望字节、校验结果。
6. 若失败：HTTP 状态码、响应片段摘要、异常堆栈。

---

## 5. B 站半截问题排查顺序（实战）

1. 先看日志是否命中 `durl` 多段或 DASH 分离。
2. 核对是否只下载了首段或只下载视频轨。
3. 检查最终文件字节数是否等于期望值。
4. 检查失败时是否返回 HTML（鉴权页/风控页）。
5. 检查请求头与 Cookie 是否完整。
6. 用 `ffprobe` 查看容器是否损坏、时长是否异常。

---

## 6. 最低可用结论

如果要避免“B 站只下载一半”，最低要求是：

1. 支持 `durl` 多段全量下载；
2. 支持 DASH 音视频合并；
3. 结束后做强校验（字节和容器）；
4. 失败时输出可定位日志，而不是仅提示“下载失败”。
