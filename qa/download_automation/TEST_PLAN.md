# 下载自动化测试方案（url.txt 批量回归）

## 1. 目标
- 对 `url.txt` 中每一行下载地址（含分享文案）执行一次端到端下载验证。
- 输出结构化测试报告（`JSON + Markdown`），用于快速定位失败项。

## 2. 范围
- 仅覆盖 `视频下载` 页签的下载能力。
- 不覆盖 `文案提取`、`AI重构`、`设置` 页签功能。

## 3. 前置条件
- 电脑已安装并可执行 `adb`。
- 手机已连接并授权 USB 调试。
- App 已安装（默认包名：`com.vdown.app`，默认 Activity：`com.vdown.app/.MainActivity`）。
- 手机保持解锁，避免系统弹窗遮挡。

## 4. 执行流程
1. 读取 `url.txt`，去掉空行后逐条测试。
2. 每条用例前清理 `logcat`，记录下载目录基线（`/sdcard/DCIM/v-down`）。
3. 通过 `adb am start` 发送 `ACTION_SEND` 文本到 App（文本可含噪声分享文案）。
4. 用 `uiautomator dump` 定位“开始下载”按钮并点击。
5. 轮询日志与目录变化，判定 `PASS / FAIL / TIMEOUT`。
6. 汇总生成：
   - `report.json`（机器可读）
   - `report.md`（人工可读）

## 5. 判定规则
- `PASS`：
  - 日志出现 `download success` 或 UI 成功关键字（例如“下载完成”），或
  - 检测到下载目录新增文件且未出现失败日志。
- `FAIL`：
  - 日志出现 `download failed` / `下载失败`。
- `TIMEOUT`：
  - 在超时时间内既未成功也未失败。

## 6. 报告字段
- 用例序号、原始输入文本、提取到的 URL、状态、耗时秒数
- 失败原因摘要、新增文件列表（若有）
- 关键日志片段（尾部截断）

## 7. 参数化能力
- URL 文件路径（默认 `url.txt`）
- 用例超时、轮询间隔
- `adb serial`（多设备场景）
- 包名、Activity
- 用例数量上限（`--limit`）
- `--dry-run`（仅解析 URL，不执行下载）

## 8. 风险与限制
- 某些平台链接强依赖 Cookie 时效，可能失败但属于环境问题。
- 若 App 弹出权限窗/系统提示遮挡按钮，会影响自动点击。
- 下载速度与网络波动会影响超时结果，可调大 `--timeout`。
